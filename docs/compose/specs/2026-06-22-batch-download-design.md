# 批量下载笔记和原文功能设计

## [S1] 问题描述

当前上传记录页面（VideoListPage）只支持单个视频的操作，没有批量下载功能。用户需要能够选择多个视频，批量下载它们的笔记和原文，提高工作效率。

## [S2] 解决方案概述

在上传记录页面添加批量下载功能，包括：
1. 前端：添加复选框选择和全选功能，顶部添加下载按钮
2. 后端：新增批量下载API端点，生成包含所有选中视频笔记和原文的ZIP文件

## [S3] 前端设计

### 3.1 选择界面
- 在每个视频项前添加复选框
- 页面顶部添加全选复选框
- 选中视频时显示选中数量
- 下载按钮显示在页面顶部，带选中数量标签

### 3.2 交互流程
1. 用户通过复选框选择视频（或使用全选）
2. 点击下载按钮
3. 前端调用后端批量下载API
4. 下载完成后显示成功提示

### 3.3 状态管理
- 添加`selectedVideos`状态，存储选中的视频ID
- 添加`isDownloading`状态，控制下载按钮的加载状态
- 添加`downloadProgress`状态，显示下载进度（可选）

## [S4] 后端设计

### 4.1 API端点
- **端点**: `POST /api/videos/batch-download`
- **请求体**: `{ "video_ids": ["id1", "id2", ...] }`
- **响应**: ZIP文件流

### 4.2 处理流程
1. 验证请求参数
2. 从数据库获取视频信息
3. 验证视频状态（必须是completed状态）
4. 生成ZIP文件：
   - 遍历每个视频
   - 获取笔记内容（content_md）
   - 获取转写原文（transcript）
   - 添加到ZIP文件中
5. 返回ZIP文件流

### 4.3 文件命名规则
- 笔记文件：`{video_filename}.md`
- 原文文件：`{video_filename}.txt`
- ZIP文件：`batch_download_{timestamp}.zip`

## [S5] 数据结构

### 5.1 请求数据
```typescript
interface BatchDownloadRequest {
  video_ids: string[];
}
```

### 5.2 响应数据
- 成功：ZIP文件流（Content-Type: application/zip）
- 失败：错误信息JSON

## [S6] 错误处理

### 6.1 前端错误处理
- 未选择视频时，下载按钮禁用
- 下载失败时显示错误提示
- 网络错误时显示重试按钮

### 6.2 后端错误处理
- 视频不存在：返回404
- 视频状态不是completed：返回400
- 生成ZIP失败：返回500

## [S8] 部署考虑

- 后端需要安装`zipfile`库（Python标准库）
- 前端需要更新API客户端
- 无需数据库迁移

## [S9] 性能考虑

- 大量视频下载时，ZIP文件可能很大
- 考虑添加下载限制（如最多50个视频）
- 考虑添加下载进度显示

## [S10] 安全考虑

- 验证用户只能下载自己上传的视频
- 验证视频状态必须是completed
- 限制批量下载的视频数量
